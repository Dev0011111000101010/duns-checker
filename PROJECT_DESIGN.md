## 🤖 ИНСТРУКЦИЯ: КАК ПОСТРОЕН ЭТОТ ДОКУМЕНТ

**Назначение:** INDEX/TABLE OF CONTENTS — быстрая навигация по проекту (не энциклопедия!)
**Целевая аудитория:** AI в первую очередь, человек во вторую (читаемость сохранена)

### ✅ ЧТО СОДЕРЖИТ этот документ:

1. **Полные пути к файлам**
   - Относительные от корня проекта
   - Пример: `backend/core/scripts/take_full_screenshot.py`

2. **Назначение файла**
   - Краткое описание (1-2 предложения)

3. **Импорты**
   - Все import в файле (внешние + внутренние)
   - Показывает взаимозависимости

4. **Список функций (def) и классов — AI-оптимизированный формат**
   - **Простые функции (1-2 параметра):** `func(param: type, default=value)` — описание
   - **Сложные функции (3+ параметра):** Заголовок + сигнатура с типами + описание + пример вызова
   - **Типы параметров:** Указаны в сигнатуре (str, bool, int, None, etc.)
   - **Примеры вызова:** Только для сложных функций

5. **Указатели на детали**
   - Где искать детальную информацию
   - Пример: "Цветовая схема: см. gui_theme.py"

### ❌ ЧТО НЕ СОДЕРЖИТ (детали остаются в коде):

- Детальные примеры кода
- Константы (COLORS = {...}, CONFIG = {...})
- Большие словари/списки
- Примеры JSON структур
- Пошаговые инструкции с кодом
- Секции "Параметры:", "Возвращает:" (избыточно для AI)
- Описание каждого параметра (если название ясное)

**При создании или изменении любого файла** → обновить этот документ.
**Если работаю с UI** → читаю детали из соответствующего файла темы/стилей.
**Если работаю с логами** → читаю форматы из logger файла.

**Принцип:** Этот документ = карта проекта. Детали читаются из файлов по требованию.

**Формат функций:**

**Стек:** Python 3.12 + Playwright (sync API) + Chromium (headless=False)

**Запуск:** `.venv/Scripts/python checker.py`

**Целевая компания:** FIBER LLC, Austin, TX, 78731 (5900 BALCONES DR STE 100)

## КЛЮЧЕВЫЕ КОМПОНЕНТЫ

### checker.py
**Назначение:** Открывает D&B Business Directory по прямому URL, читает таблицу результатов, проверяет наличие FIBER LLC (Austin, TX). Браузер остаётся открытым после завершения — закрывается вручную через Enter.

**Импорты:**
```python
from playwright.sync_api import sync_playwright, TimeoutError as PlaywrightTimeoutError
```

**Константы:**
- `SEARCH_URL` — прямой URL поиска с параметрами в hash-фрагменте
- `TARGET_NAME = "FIBER LLC"` — искомое название компании
- `TARGET_CITY = "Austin"`, `TARGET_STATE = "TX"` — целевая локация

**Функции (def):**
- `check_fiber_llc() -> bool` — весь цикл: запуск браузера → проверка рендеринга → загрузка D&B → чтение таблицы → вывод итогов. Возвращает `True` если целевая компания найдена.

**Exit codes:** `0` — компания найдена, `1` — не найдена, `2` — ошибка (пустая таблица / таймаут)

**Детали:** см. checker.py
